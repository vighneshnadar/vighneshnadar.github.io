How to connect to AWS EC2 Windows Server 2019 Instance?

12 December, 2021 | Vighnesh | AWS

Let's try to understand Elastic IP in a different way.What if there was no term such as Elastic IP?What's the problem?

Suppose you want to host a website and you have a domain name which points to the public IP of your server,if you are using AWS EC2 it's the public IP of your instance which has your server running.If for any reason you switch off the machine or restart the machine in which your server is running ,it may allocate different public IP.

Ohh No! But you pointed your domain name to the public IP provided to you and it has been released in the IP pool.Now your server is running,but it has a different public IP this time.Now you again need to point your domain name to this new IP.Ufff!

For the rescue comes Elastic IP.It is static unlike public IP which is dynamic.When you associate an elastic IP with your server it never changes it is reserved for you until and unless you disassociate it.Now when you again point your domain name to this Elastic IP the problem is solved.

Let's take the AWS instance created in the previous post http://127.0.0.1:8000/post/getting-started-aws-ec2-basics-how-create-instance/ and try to associate an Elastic IP to it.

Here as you can see this is my current public IP  -35.154.20.103

 

Now from the sidebar click Elastic IP and then click on Allocate Elastic IP.

 

If you are allocating a Elastic IP for your account it should be attached to some instance if not it incurrs charges as of 2020.

Now we have allocated a new elastic IP so we have to associate it with a instance.Lets associate it with the instance we created earlier,Example Project.

Here we select the instance with which we want our Elastic IP to be associated.

 

We have successfully associated the Elastic IP with our instance.

Here  as you can see,earlier our IP was 35.154.20.103 which was  dynamic IP now we have associated the elastic  IP 35.154.178.122 which is static.

 

If you have a elastic IP which is not associated with any instance,you can simply disassociate and release the Elastic IP.Simply go to Action->Networking->Disassociate Elastic IP.address.

Now Release the IP address so that it's no longer associated with your account.By going to Elastic IP from sidebar.Actions->Release.
